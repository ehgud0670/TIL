## 기본 내용

* 자료형(타입 , type )
  * 정적 타입 : 타입(type), 자료형을 컴파일 시에 결정하는 것. Java가 정적 타입 언어. 
  * 동적 타입  : 타입(자료형)을 컴파일 시 자료형을 정하는 것이 아니고 , 실행시에 결정하는 것. Python이 동적 타입 언어. 
    * 덕 타이핑 : 동적 타이핑의 한 종류로, 객체의 변수 및 메소드의 집합이 **객체의 타입을 결정하는 것(런타임)** 을 말한다.
* DRY : Don't Repeat Yourself : 반복해야할 특정 코드가 있다면 메소드를 만들어 처리하라
* 업 캐스팅(자식이 부모로 형변환) 후에도 재정의(Overriding)된 메소드가 있다면 재정의된 메소드가 호출된다. 
* String은 불변 객체이다. ( 상태가 변경되어야 한다면 그 상태를 가지는 새로운 객체를 생성해야 한다. )
* 자식은 반드시 기능을 제공해야 하고 부모가 구현을 제공할 필요가 없다면 , 추상(abstract) 메소드를 사용하는 게 좋다. 
* 리팩토링(Refactoring) : 기존 코드의 동작을 변경하지 않고 구조를 개선하는 작업 <br> 마틴 파울러 : 코드의 유지보수성을 떨어트리는 요소, 냄새(smells)를 제거 => Replace type code with polymorphism
* AOP(관점 지향 프로그래밍) :  Aspect Oriented Programming => 어노테이션을 사용하는 두번째 이유. 즉 반복적으로 생기는 보일러플레이트를 제거할 수 있다. 좀 더 선언적으로 표현이 가능하게 한다. (@Test, @build)
* Flyweight Pattern : 동일한 내용의 객체를 공유할 수 있다.
<br>=> 플라이웨이트 패턴은 비용이 큰 자원을 공통으로 사용할 수 있도록 만드는 패턴이다. (팩토리 패턴 사용)
  
  * 장점 
    * 많은 객체를 만들 때 성능을 향상시킬 수 있다. 
    * 많은 객체를 만들 때 메모리를 줄일 수 있다.
    * state pattern 과 쉽게 결합될 수 있다. 

  * 단점
    * 특정 인스턴스의 공유 컴포넌트를 다르게 행동하게 하는 것이 불가능하다. (개별 설정이 불가능하다.)

* COW (Copy On Write) : 방어적 복사 
* == : 참조 동등성을 판단한다. 
* equals : 객체 동등성을 판단한다. 
* interface는 클래스 간 느슨한 결합 관계를 만들기 위해 사용한다. **완벽한 추상 클래스**라고 할 수 있다. 그러나 불완전한 메서드로만 이루어져 있다. 따라서 **인스턴스화 할 수 없다.** 
* protected 를 사용하면 안되는 이유는 재사용이 불가능하기 때문이다. => 이해가 더 필요하다.  
* 오버헤드 (Overhead)란 어떤 처리를 하기 위해 들어가는 간접적인 처리 시간, 메모리등을 말한다. 
* 상태에 따라 만들려면 enum을 사용해야 한다. 
* FileInputStream , FileOutputStream(바이트스트림)은 바이트(Byte) 형식의 데이터를 읽고 쓴다. 
* FileReader, FileWriter(문자스트림)은 문자단위(char 배열 or String) 형태의 문자를 읽거나 쓴다. 
* Vector는 무조건 동기화이기 때문에 **단일 스레드에서 성능이 떨어지고 멀티 스레드에서도 이제 사용하지 않는다.**
* 단일 스레드 처리에는 ArrayList를 쓰고, **동기화가 필요할 때에는 Collection, synchronizedCollection, synchronizedList , Map 을 사용한다.**
